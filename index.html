Домашнє завдання: GIT

  • git config --global user.name “name” - встановити ім'я користувача
  • git config --global user.email “email@google.com” - встановити email користувача
  • git init - створити новий репозиторій
  • git status - допомагає оприділити, які файли та в якому стані знаходяться
  • git add - додає всі змінені файли
  • git commit -m - створити новий коміт
  • git log - показує історію комітів
  • git checkout - багатофункціональна програма, один з варіантів це переключення на гілку
  • git branch - створення гілки
  • git push - завантажує всі зміни у віддалений репозиторій
  • git pull - Команда, що використовується для оновлення локальної версії сховища з віддаленого
  • git merge - об'єднює гілки в одну
  • git clone - клонує віддалений репозиторій

gitignore служить для вказівки в інших файлах і папках, які необхідно приховати від системи контролю версії git. Як правило, приховують файли або папки, які автоматично генерують вміст, або вводять конфігураційні параметри, які можуть відрізнятися між собою, хто спільно працює над проектом.

  • git revert - відкочується Комміт шляхом створення нового, зміни в якому протилежні тому коммітов, який потрібно скасувати
  • git reset - виконує скасування змін
  • git rebase - дозволяє згрупувати всі коміти в один патч і перенестиміж гілками
  • git cherry-pick - дозволяє перенести конкретний коміт з одної гілки в іншу, без використання git merge;
  • git stash дозволяє зберегти зміни у поточному файлі в окреому сховищі, а сам файл повернути до початкого стану.
